数据结构

  • 数据结构：相互之间存在一种或多种特定关系的数据元素的集合。

  • 数据元素：数据的基本单位，通常作为整体，由若干数据项组成

  • 数据项：组成数据元素的具有独立含义的最小单位
  • 数据结构三元素：
    • 逻辑结构：数据元素之间的逻辑关系。包括集合结构、线性结构、树形结构、图形结构
    • 存储结构：数据元素在计算机存储中的表示。包括顺序存储结构、链式存储结构
    • 数据的运算：对数据元素可以施加的操作

三元素

逻辑结构

  • 集合结构
  • 线性结构
  • 树形结构（线性结构是特殊的树形结构）
  • 图形结构

存储结构

存储结构既要反应数据元素本身，也要反应数据元素之间的关系

  • 顺序结构

  • 链式结构

    //线性链式存储
struct Node{
    int data;  //4字节
    Node* next;//4字节
}

//树型链式存储
struct Node{
    int data; 
    Node* lChild; //左孩子地址
    Node* rChild; //右孩子地址
}

图的表示：邻接矩阵、邻接表，皆可使用顺序或链式存储

运算

建立、清除、求长、判空、判满、获取、更新、插入、删除、查找、遍历

数据类型

一组性质相同的值的集合及定义在此集合上的一些操作的总称

例：C 中的基本数据类型：int、float、char、double、void

抽象数据类型

数据结构的表现形式：

  1. 逻辑抽象模型（抽象数据类型 ADT，如链表 List）

  2. 具体实现（C，java······）

抽象数据类型可用三元组表示：ADT = (D,S,P)

算法

复杂度

//插入排序
//总体最多执行n-1次，每次最多执行n-1次，平方复杂度
//最少执行n-1次，每次最少执行1次，线性复杂度
//平均复杂度为O(n^2)，通常与最坏情况相同
void InsertSort(int data[], int n){
  int buffer = 0;
  for(int i = 1; i < n; i++){
    buffer = data[i];
    int j = i - 1;
    while(j >= 0 && data[j] > buffer){
      data[j + 1] = data[j];
      j--;
    }
    data[j + 1] = buffer;
  }
}

算法复杂度层次：
$O(1) < O(logn) < O(\sqrt{n}) < O(n) < O(nlogn) < O(n^2) < O(n^3) < O(2^n) < O(n!)$

递归

//减治
int SumI(int A[], int n)
{
  return n < 1 ? 0 : sumI(A, n-1) + A[n-1]
}
//时间复杂度 O(n)

//分治
int sum(int A[], int lo, int hi){
  if(lo == hi){
    return A[lo];
  }
  int mi = (lo + hi)/2;
  return sum(A, lo, mi) + sum(A, mi, hi)
}
//时间复杂度 O(n)

//斐波那契，递归存在大量重复调用
int fib(int n){
  return (n < 2) ? n : fib(n-1) + fib(n-2);
}
